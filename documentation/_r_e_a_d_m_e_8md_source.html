18 ## How To Use
19 
20 This project has been built largely to be imported as a managed-code plugin. Within a Unity project, navigate to `${project_root}/Assets/Plugins`, and clone this repository and set it up as a submodule:
21 
22 ```sh
23 cd ${project_root}/Assets/Plugins
24 git submodule add https://github.com/UCRBrainGameCenter/BGC_Tools.git
25 ```
26 
27 If you're not familiar with using git submodules, [check out Trevor's blog post on submodules](https://tstavropoulos.github.io/blargs/git_submodules.html).
28 
29 Take a look at our MIT Licensed [CleanBGCProject Unity Demo Project](https://github.com/UCRBrainGameCenter/CleanBGCProject) to see some of these tools in action.

31 ## Features
32 
33 ### User Management
34 
35 - Easily-integrated system for managing multiple distinct user profiles in `Users.PlayerData`
36 - User data is serialized into JSON and readily available for inspection, debugging, reproduction
37 
38 ### Audio
39 
40 - Advanced Audio synthesis library (Check out examples in `Tests.SynthesisTests`)
41 - Fourier transforms, Spectrogram generation
42 - Managed code to Read and Write WAV files in `Audio.WaveEncoding`
43 - Midi Encoding, Decoding, and Rendering (Alpha) in `Audio.Midi.MidiEncoding`
44 
45 ### Custom Data Structures
46 
47 - `IDepletable` containers, representing sets of values that can be depleted when taken, but can also be refreshed (located in `DataStructures.Generic`)
48  - `DepletableList<T>` will return values from an underlying collection in a fixed order, and can be refreshed to its original state (like a stack that can be reset)
49  - `DepletableBag<T>` will return values in a random order from an underlying collection (like shuffle on a music player)
50 - Efficient `RingBuffer<T>` implementation
51 - Several forms of `IPool`, for pooling objects
52 
53 ### Mobile, Desktop, and Editor-Friendly File I/O
54 
55 - Streamline the process of accessing data directories and subdirectories on all platforms with `IO.DataManagement`
56 - Convenient JSON reading, parsing, and writing with `IO.FileReader` and `IO.FileWriter`
57 - Zip Compression/Decompression support
58 - Inclusion of [LightJson](https://github.com/MarcosLopezC/LightJson)
59 
60 ### Mathematics
61 
62 - Robust implementation of Complex number support with `Mathematics.Complex32` and `Mathematics.Complex64`
63 - Fast Fourier Transforms (including for sample buffers that are not a power of 2)
64 - Bezier Curves
65 - Implementation of common math functions in `Mathematics.GeneralMath`
66 - Number factorization, least common multiple, and combinatorics
67 - Numeric Recipe for the Modified Bessel Function of the First Kind
68 
69 ### State Machine
70 
71 - Fast, fully-programmatic state machine implementation in `StateMachine` namespace.
72 - Sample implementation included in [README.md](StateMachine/README.md).
73 
74 ### UI
75 
76 - Multi-featured, convenient Modal Dialog in `UI.Dialogs.ModalDialog`
77 - ModePanels for control of UI Frame Instances in `UI.Panels.ModePanel`
78 
79 ### General Utilities
80 
81 - Robust Semantic Versioning struct in `Utility.ApplicationVersion`
82 - Coroutine Management utility in `Utility.CoroutineUtility`
83 - Local file-browser Scene and Scripts, to allow the inspection of application files on-device (`UnityAssets/Scenes/FileBrowser.unity`)
